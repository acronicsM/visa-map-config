# Ключевые архитектурные решения

## Почему PostGIS
Геометрия стран хранится в PostGIS, ST_SimplifyPreserveTopology
уменьшает размер GeoJSON с ~50MB до ~5MB без потери качества.

## Почему два запроса к restcountries API
API отдаёт 400 если запросить больше ~8 полей за раз.
Решение: два запроса (основные данные + языки/домены) с объединением по cca2.

## Почему unary_union для геометрии
Natural Earth shapefile содержит несколько записей для одной страны
(Австралия = континент + острова Эшмор + Коралловое море + Индийский океан).
Без union берётся последняя запись (маленький остров).

## Почему promoteId вместо generateId в MapLibre
generateId нельзя использовать вместе с promoteId.
promoteId: 'iso2' позволяет использовать строковые id для feature-state,
что нужно для hover эффекта и setPaintProperty по iso2.

## Почему confidence_level=3 по умолчанию
Passport Index данные могут отставать на 1-3 месяца.
Уровень 3 сигнализирует пользователю что данные не верифицированы.

## Почему Set Cover для языков
195 стран — у каждой 1-4 официальных языка.
Greedy Set Cover находит минимальный набор языков покрывающий все страны.
Результат: 154 языка покрывают 249 из 250 стран.
primary_language используется для поиска локальных источников.

## Почему source_type в rss_sources
news_agency: нужна фильтрация по keywords
aggregator: URL уже содержит поисковый запрос, все новости релевантны
official: официальные источники, все новости релевантны
Это позволяет одному монитору работать с принципиально разными источниками.

## Имя компонента VisaMap вместо Map
Map — встроенный JavaScript объект.
Next.js путался при импорте компонента с таким именем.
Переименовано в VisaMap.

## Импорт безопасности: Redis + Postgres, без колонки балла в БД
Числовой **`safety_final_score`** хранится в Redis (ключ `countries:safety_final_scores:v1`,
без TTL до следующей загрузки), чтобы не плодить миграции под «сырой» балл.
Для фильтров и GeoJSON используется **`countries.safety_level`** (`safe` / `unsafe` / `dangerous`),
вычисляемый при импорте по порогам `SAFETY_SCORE_SAFE_MIN` / `SAFETY_SCORE_UNSAFE_MIN`.

Массовое обновление уровней в БД делается **циклом ORM `update(Country)` по iso2**, а не одним
`UPDATE … FROM jsonb_to_recordset(CAST(:payload AS jsonb))`: с **asyncpg** такой сырой запрос
часто обновлял **0 строк** при успешном выполнении, из-за чего Redis и Postgres расходились.

После успешного импорта вызывается **`cache_delete(countries:geodata:v2)`**, иначе карта могла
долго отдавать старые `safety_level` из Redis-кеша GeoJSON (TTL 24h).
